# 🗺️ 项目演进路线图 (Development Roadmap)

## 🏃 Sprint 1: 单机文字版基础 (Terminal MVP)

**【核心目标 (Goal)】**
搭建游戏最基础的底层运行逻辑，实现终端交互框架，完成空间拓扑数据结构的构建与玩家的基础定位移动。

**【功能列表 (Backlog)】**
* **引擎基建**：构建主循环 (Main Loop) 与基础的终端 I/O 指令解析器 (REPL)。
* **地图系统**：设计房间 (Room) 的数据结构及地图图状拓扑连通性。
* **角色系统**：初始化玩家实体（坐标、基础属性定义）。
* **基础指令**：实现 `look` (查看四周环境) 与 `go [n/s/e/w]` (方向移动) 等核心基础命令。

**【验收标准 (Definition of Done)】**
* [ ] 程序能在本地终端稳定运行，具备输入容错处理（不因异常输入而 Crash）。
* [ ] 玩家可以在至少包含 3 个相互连接的房间节点中自由导航。
* [ ] 每次移动后，控制台能正确打印当前房间的描述文本和可用出口。

## ⚔️ Sprint 2: 单机游戏性完善 (Core Gameplay Loop)

**【核心目标 (Goal)】**
赋予虚拟世界生命力与交互性，引入实体资源获取（Loot）以及核心战斗机制，闭环单机游戏体验。

**【功能列表 (Backlog)】**
* **物品与背包系统**：定义装备/消耗品数据结构，实现背包容量逻辑与 `take`/`drop`/`inventory` 指令。
* **NPC 与敌对实体**：在地图特定节点生成静态 NPC 和环境生物 (Monsters)。
* **基础交互**：实现与非敌对 NPC 的文本交互 (`talk` 指令)。
* **战斗机制**：引入基于回合制或敏捷度的数值对抗系统（HP/Attack/Defense），实现 `attack` 指令。

**【验收标准 (Definition of Done)】**
* [ ] 玩家可以拾取环境中的物品存入背包，并在不同房间丢弃。
* [ ] 玩家能够向敌对生物发起攻击，终端有明确的战斗结算反馈（如伤害数值、怪物掉落）。
* [ ] 玩家生命值归零时，正确触发 `Game Over` 流程并重置状态。

## 🌐 Sprint 3: 网络化重构 (Multiplayer & C/S Architecture)

**【核心目标 (Goal)】**
进行底层架构的重大重构，从单机循环迁移至完整的 C/S (Client/Server) 架构，实现多玩家物理可见性与状态同步。

**【功能列表 (Backlog)】**
* **网络基建**：基于 Socket (TCP) 搭建稳定的服务端监听与客户端连接机制。
* **并发处理**：引入多线程或异步 I/O (如 Select/Epoll) 管理多客户端的并发请求。
* **权威服务端**：服务端接管所有地图校验和战斗逻辑（Server-Authoritative），防止客户端作弊。
* **事件广播机制**：实现房间内的状态广播（如：“玩家 A 走进了房间”、“玩家 B 拔出了剑”）。

**【验收标准 (Definition of Done)】**
* [ ] 至少支持 2 个独立客户端同时稳定连接到服务端。
* [ ] 客户端 A 的移动或状态改变，能被同房间的客户端 B 实时在终端中看到。
* [ ] 妥善处理客户端意外断开连接（掉线）的异常，保证服务端进程不崩溃。

## 💾 Sprint 4: 数据持久化与账户系统 (Data Persistence)

**【核心目标 (Goal)】**
接入持久化存储方案，保障玩家的努力（装备、等级、位置）得以长久保存，实现跨会话的游戏体验。

**【功能列表 (Backlog)】**
* **数据库集成**：引入关系型数据库 (如 MySQL) 或轻量级 DB (SQLite) 进行数据落盘。
* **账户鉴权系统**：实现玩家账户注册 (Register) 与加密登录 (Login) 验证流程。
* **状态序列化**：设计数据库表/文档结构，实时或定期保存玩家对象（属性、背包、当前坐标）。
* **会话恢复**：玩家登录后自动读取数据库状态，无缝恢复至下线前的场景和状态。

**【验收标准 (Definition of Done)】**
* [ ] 玩家可以使用唯一账号密码注册，且密码在数据库中经过哈希加密存储。
* [ ] 玩家在房间 A 拾取物品 B 后主动下线（或强退），重新登录时依然在房间 A 且背包中拥有物品 B。
* [ ] 能够处理并发状态下的数据库读写冲突（基础的事务锁机制）。

## 🎨 Sprint 5: Web 前后端分离重构 (Modern Web API & UI)

**【核心目标 (Goal)】**
彻底脱离命令行黑框，拥抱现代 Web 生态，提供友好的图形化 UI 和跨端访问能力。

**【功能列表 (Backlog)】**
* **API 层封装**：将原有的 Socket 纯文本协议升级为 RESTful API 或 WebSockets，以提供标准接口。
* **前端工程化基建**：搭建 Web 客户端页面（HTML/CSS/JS 或引入现代前端框架）。
* **图形化 UI 设计**：实现可视化的状态面板（HP/MP条）、交互式背包界面以及动作指令按钮。
* **即时通讯反馈**：在网页端实现基于 WebSockets 的游戏事件滚动日志和公共聊天频道。

**【验收标准 (Definition of Done)】**
* [ ] 玩家无需配置任何本地环境，仅需通过浏览器访问指定 URL 即可顺畅游玩。
* [ ] 游戏指令可通过点击页面 UI 按钮触发，完全取代纯键盘指令输入。
* [ ] 前端页面能够实时响应后端推送的数据，并无刷新地更新画面状态。
